# Data Directory

## Structure

```
data/
├── raw/                      # Original, unmodified data (DVC-tracked)
│   └── simulation_logs/      # Raw episodes from simulator
├── processed/                # Cleaned, normalized data
│   ├── images/               # Resized camera frames
│   └── labels/               # Segmentation masks, bounding boxes
├── features/                 # Engineered features
│   ├── embeddings/           # Perception model encodings
│   └── state_vectors/        # Normalized robot states
└── README.md
```

## Workflow

1. **Raw Data**: Generated by `simulation/generate_data.py`
   - Stores complete episode logs with images, states, actions, rewards
   - Version controlled with DVC
   
2. **Processed Data**: Created by `pipelines/data_pipeline.py`
   - Resized images, normalized labels
   - Train/val/test splits
   - Ready for model training
   
3. **Features**: Generated during training
   - Vision embeddings (perception model outputs)
   - State vectors for RL
   - Used for analysis and monitoring

## Data Collection Strategy

- **Simulation**: Generate diverse episodes covering various scenarios
- **Real World** (future): Collect actual robot trajectories
- **Active Learning**: Request human labels on uncertain predictions

## Storage & Access

For local development:
```bash
# Generate simulation data
python simulation/generate_data.py --num_episodes 1000

# Process raw data
python pipelines/data_pipeline.py
```

For production (DVC):
```bash
dvc pull  # fetch remote data
dvc push  # push to remote storage
```

## Versioning

Track data versions with DVC:
```bash
dvc add data/raw/
git add data/raw.dvc .gitignore
git commit -m "Update raw data v2.1"
```
